안녕하세요! 오늘 **[텔레그램 자가 업데이트(Self-Update) 봇 구축]** 시리즈에 이어, 오늘은 퀀트 자동매매 서버 인프라의 화룡점정을 찍을 심화편, **[독립형 롤백(Rollback) 구조대 봇 구축]**을 소개합니다.

단순히 안정된 패치만 받아서 사용하시는 일반 사용자분들이라면 지난번 자가 업데이트만으로도 충분합니다. 하지만 스스로 코드를 수정하고 새로운 전략을 테스트하는 **'개발자급 유저'**라면 한 번쯤 이런 끔찍한 경험을 해보셨을 겁니다.

> *"야심 차게 코드를 수정해서 깃허브에 올리고 /update를 쳤는데... 아뿔싸, 오타가 있어서 봇이 뻗어버렸다! 텔레그램은 먹통이 되었고, 밖인데 결국 PC를 켜서 SSH 터미널로 서버에 접속해야만 했다..."*
> 
메인 봇이 죽으면 메인 봇에게 "다시 켜져라"라고 명령을 내릴 수 없습니다. 이것이 바로 **단일 장애점(SPOF)**입니다. 대형 IT 기업들은 이 문제를 해결하기 위해 메인 서버가 죽었을 때 외부에서 강제로 심폐소생술을 하는 **'독립된 감시/구조 서버'**를 둡니다.

오늘은 이 엔터프라이즈급 아키텍처를 우리의 퀀트 봇에 그대로 이식해 보겠습니다. 오직 내 명령만 듣는 철통 보안의 **'구조대 봇'**을 별도로 만들어, 밖에서도 핸드폰 하나로 3초 만에 봇을 과거의 정상 상태로 부활시키는 마법을 구현해 봅시다!

## ⚔️ [1단계] 메인 봇의 '안전띠' 결속 (첨부파일 적용)
먼저 메인 봇이 업데이트를 하러 가기 전에, **"현재 정상 작동 중인 코드"**를 스스로 백업해 두는 유서(안전띠) 작성 로직이 필요합니다.
게시글에 첨부된 **plugin_updater.py** 파일을 다운로드하여 메인 봇 폴더에 덮어쓰기 해주십시오.
이제 메인 봇은 /update 명령을 받으면 가장 먼저 stable_backup 이라는 백업 폴더를 만들고, 현재 정상 작동하는 코드들을 안전하게 복사해 둔 뒤 업데이트를 진행하게 됩니다.

## 🛡️ [2단계] 구조대 봇 전용 비밀 벙커 및 금고 생성
구조대 봇은 메인 봇과 한 폴더에 섞이면 안 됩니다. 메인 봇 폴더 바깥(/home)에 완전히 독립된 안전가옥(폴더)을 만듭니다.

*(※ 아래 명령어들에서 [본인봇이름] 부분을 실제 사용 중인 봇 이름으로 바꿔서 입력하세요!)*

# 1. 메인 폴더 밖으로 이동하여 구조대 벙커 생성
cd /home
mkdir [본인봇이름]_rescue
cd [본인봇이름]_rescue

그다음, 텔레그램 BotFather에게 가서 **새로운 봇(구조대 전용 봇)**을 하나 더 만들고 토큰을 발급받습니다.

아래 명령어를 터미널에 입력하여 봇 토큰과 본인의 텔레그램 고유 ID, 그리고 살려낼 메인 봇의 데몬 이름을 비밀 금고(.env)에 안전하게 봉인합니다. (아무나 롤백을 치지 못하도록 **화이트리스트 접근 통제**를 하기 위함입니다.)

cat << 'EOF' > .env
# 🚨 구조대 봇 토큰 및 사령관 ID (절대 유출 금지)
RESCUE_BOT_TOKEN="여기에_새로_발급받은_구조대_토큰_입력"
ADMIN_CHAT_ID="본인의_숫자_텔레그램_ID"
daemon_name="[본인봇이름]" # 살려낼 메인 봇의 데몬(서비스) 이름
EOF

## 🚑 [3단계] 불사조 구조대 엔진 코드 다운로드 및 배치
이제 구조대 봇의 핵심 엔진을 심을 차례입니다. 이 봇은 오직 지정된 사령관(ADMIN_CHAT_ID)의 명령만 들으며, /rescue 명령이 떨어지면 stable_backup의 파일을 메인 봇으로 덮어씌우고 강제 재가동(restart) 시킵니다.
 1. 게시글에 첨부된 **rescue_bot.py** 파일을 다운로드합니다.
 
  2. 방금 만든 구조대 벙커 폴더(/home/[본인봇이름]_rescue/) 안에 해당 파일을 넣어주십시오.
   *(💡 팁: 첨부된 파일 내부의 MAIN_BOT_DIR 변수 경로가 본인의 실제 메인 봇 폴더 경로와 일치하는지 꼭 확인 후 수정해 주십시오.)*

## 🧟‍♂️ [4단계] 구조대 봇을 절대 죽지 않는 좀비 데몬으로 만들기
구조대 봇이 터미널 창을 닫아도 365일 백그라운드에서 대기하도록, 리눅스 OS의 심장(systemd)에 좀비 데몬으로 등록합니다.

sudo tee /etc/systemd/system/[본인봇이름]_rescue.service > /dev/null << 'EOF'
[Unit]
Description=Rescue Bot (Immortal Zombie Mode)
After=network.target

[Service]
User=root
WorkingDirectory=/home/[본인봇이름]_rescue
ExecStart=/usr/bin/python3 /home/[본인봇이름]_rescue/rescue_bot.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 데몬 등록 및 재부팅 시 자동 실행 락온

sudo systemctl daemon-reload
sudo systemctl enable [본인봇이름]_rescue
sudo systemctl start [본인봇이름]_rescue

## 🎯 완벽한 무결점 아키텍처의 완성
수고하셨습니다! 이제 여러분의 자동매매 시스템은 메인 봇이 뻗어버리는 최악의 상황에서도 절대 무너지지 않습니다.
새로운 전략을 개발하다가 봇이 먹통이 되었나요? 당황해서 노트북을 열고 SSH에 접속할 필요가 없습니다. **핸드폰 텔레그램을 켜고, 구조대 봇에게 /rescue 단 한 번만 입력하십시오.** 3초 만에 봇은 과거의 완벽했던 상태로 타임머신을 타고 되돌아옵니다.
개발의 두려움을 완전히 없애주는 100% 무결점 이중화 클러스터 구축, 지금 바로 여러분의 서버에 적용해 보십시오!
